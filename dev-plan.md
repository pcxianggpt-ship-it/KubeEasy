# KubeEasy 开发计划

> **版本**: v1.1
> **日期**: 2025-11-15
> **项目**: K8s 离线一键部署框架
> **基于文档**: requirement.md, KubeEasy技术文档.md

## 1. 项目概述

### 1.1 目标
将现有的离线安装 shell 脚本改造成一套**可配置、可重入、支持并行/串行流程控制、带规范化日志和状态查询**的全流程自动化安装框架。

### 1.2 核心特性
- **离线部署**: 完全不依赖外网，支持多架构（amd64/arm64）
- **并发控制**: 智能的并行/串行任务调度
- **状态管理**: 完整的安装状态跟踪和恢复机制
- **配置驱动**: 通过 YAML 配置文件控制整个安装流程
- **多发行版兼容**: 支持 CentOS/RHEL、Kylin、Ubuntu 等
- **NFS 支持**: 内置 NFS 依赖安装和配置
- **IPv6 双栈**: 完整的 IPv4/IPv6 双栈网络支持

## 4. 核心功能重点

### 4.1 NFS 依赖安装功能 ⭐
**目标**: 在安装依赖阶段自动安装 NFS 相关依赖包

**实现要点：**
- 优先使用 `yum` 安装 NFS 依赖（nfs-utils, rpcbind）
- 如果 `yum` 失败，自动切换到离线 RPM 包安装
- 完善的错误处理和状态跟踪
- 支持多种 Linux 发行版

**测试验证：**
- 离线环境下 NFS 依赖安装测试
- 不同发行版兼容性测试
- 安装失败时的降级策略测试

### 4.2 Registry 安装优化 ⭐
**目标**: 按照现有脚本实现 registry 安装，支持压缩包中的配置文件

**实现要点：**
- 确保 Docker 先安装并运行
- 使用压缩包中的 config.yml 文件
- 保持与 installscript/03.registry_install.sh 一致
- 支持认证和非认证两种模式

### 4.3 IPv6 双栈网络支持 ⭐
**目标**: 完整的 IPv4/IPv6 双栈网络配置支持

**实现要点：**
- 系统级 IPv6 参数配置
- 网卡 IPv6 地址配置
- Kubernetes 集群双栈初始化
- IPv6 连通性验证
